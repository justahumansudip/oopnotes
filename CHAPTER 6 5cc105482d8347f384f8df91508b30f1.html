Concept of Virtual Functions

Q: What are virtual functions? A: Virtual functions are member functions in a base class that can be redefined in a derived class. When a virtual function is called for a derived class object, the compiler identifies the object type at runtime and calls the derived class's version of the function.

Example:

class Shape {
public:
   virtual void draw() {
      cout << "Drawing a Shape" << endl;
   }
};

class Circle : public Shape {
public:
   void draw() {
      cout << "Drawing a Circle" << endl;
   }
};

int main() {
   Shape* s = new Circle();
   s->draw();
   return 0;
}

Output: Drawing a Circle

Late Binding

Q: What is late binding? A: Late binding is the process of determining at runtime which function to call based on the object's type. It is also known as dynamic binding or runtime polymorphism.

Example:

class Shape {
public:
   virtual void draw() {
      cout << "Drawing a Shape" << endl;
   }
};

class Circle : public Shape {
public:
   void draw() {
      cout << "Drawing a Circle" << endl;
   }
};

int main() {
   Shape* s = new Circle();
   s->draw();
   delete s;
   s = nullptr;
   return 0;
}

Output: Drawing a Circle

Abstract Class and Pure Virtual Functions

Q: What is an abstract class? A: An abstract class is a class that cannot be instantiated. It is used as a base class to provide an interface for its derived classes.

Q: What is a pure virtual function? A: A pure virtual function is a virtual function that has no implementation in the base class. It is used to make a class abstract.

Example:

class Shape {
public:
   virtual void draw() = 0;
};

class Circle : public Shape {
public:
   void draw() {
      cout << "Drawing a Circle" << endl;
   }
};

int main() {
   Shape* s = new Circle();
   s->draw();
   delete s;
   s = nullptr;
   return 0;
}

Output: Drawing a Circle

Virtual Destructors

Q: Why do we need virtual destructors? A: We need virtual destructors when we have a derived class object that is dynamically allocated and deleted through a pointer to the base class. If the base class destructor is not virtual, it will only call the base class destructor and not the derived class destructor, resulting in memory leaks.

Example:

class Shape {
public:
   virtual ~Shape() {
      cout << "Shape Destructor" << endl;
   }
};

class Circle : public Shape {
public:
   ~Circle() {
      cout << "Circle Destructor" << endl;
   }
};

int main() {
   Shape* s = new Circle();
   delete s;
   s = nullptr;
   return 0;
}

Output:

Circle Destructor
Shape Destructor

Virtual Base Class

Q: What is a virtual base class? A: A virtual base class is a class that is inherited virtually by multiple classes. It ensures that there is only one copy of the base class in the derived class hierarchy.

Example:

class Shape {
public:
   virtual void draw() {}
};

class TwoDShape : virtual public Shape {
public:
   virtual void draw() {}
};

class ThreeDShape : virtual public Shape {
public:
   virtual void draw() {}
};

class Sphere : public TwoDShape, public ThreeDShape {
public:
   virtual void draw() {}
};

int main() {
   Shape* s = new Sphere();
   s->draw();
   delete s;
   s = nullptr;
   return 0;
}

Output: No output (virtual function does not have a body)

Friend Function and Static Function

Q: What is a friend function? A: A friend function is a non-member function that has access to the private and protected members of a class.

Q: What is a static function? A: A static function is a member function that belongs to the class and not to a specific object. It can be called using the class name and does not require an object to be instantiated.

Example:

class Shape {
private:
   int length;
   int width;
public:
   Shape(int l, int w) : length(l), width(w) {}
   friend void display(Shape s);
   static int count;
   static void incrementCount() {
      count++;
   }
};

void display(Shape s) {
   cout << "Length: " << s.length << ", Width: " << s.width << endl;
}

int Shape::count = 0;

int main() {
   Shape s1(5, 10);
   Shape s2(15, 20);
   display(s1);
   display(s2);
   Shape::incrementCount();
   cout << "Count: " << Shape::count << endl;
   return 0;
}

Output:

Length: 5, Width: 10
Length: 15, Width: 20
Count: 1

Assignment and Copy Initialization

Q: What is assignment initialization? A: Assignment initialization is initializing an object after it has been created using the assignment operator.

Q: What is copy initialization? A: Copy initialization is initializing an object when it is created using the equals sign.

Example:

class Shape {
public:
   int length;
   int width;
   Shape() {}
   Shape(int l, int w) : length(l), width(w) {}
   Shape& operator=(const Shape& s) {
      length = s.length;
      width = s.width;
      return *this;
   }
};

int main() {
   Shape s1(5, 10);
   Shape s2;
   s2 = s1;
   Shape s3 = s1;
   cout << "S2: Length: " << s2.length << ", Width: " << s2.width << endl;
   cout << "S3: Length: " << s3.length << ", Width: " << s3.width << endl;
   return 0;
}

Output:

S2: Length: 5, Width: 10
S3: Length: 5, Width: 10

Copy Constructor

Q: What is a copy constructor? A: A copy constructor is a special constructor that creates a new object as a copy of an existing object.

Example:

class Shape {
public:
   int length;
   int width;
   Shape() {}
   Shape(int l, int w) : length(l), width(w) {}
   Shape(const Shape& s) {
      length = s.length;
      width = s.width;
   }
};

int main() {
   Shape s1(5, 10);
   Shape s2(s1);
   cout << "S2: Length: " << s2.length << ", Width: " << s2.width << endl;
   return 0;
}

Output: S2: Length: 5, Width: 10

This Pointer

Q: What is the this pointer? A: The this pointer is a pointer that points to the object that is calling the member function.

Example:

class Shape {
public:
   int length;
   int width;
   Shape(int l, int w) : length(l), width(w) {}
   void display() {
      cout << "Length: " << this->length << ", Width: " << this->width << endl;
   }
};

int main() {
   Shape s(5, 10);
   s.display();
   return 0;
}

Output: Length: 5, Width: 10

Concrete Classes

Q: What is a concrete class? A: A concrete class is a class that can be instantiated and has no pure virtual functions.

Example:

class Shape {
public:
   int length;
   int width;
   Shape(int l, int w) : length(l), width(w) {}
   void display() {
      cout << "Length: " << length << ", Width: " << width << endl;
   }
};

class Circle : public Shape {
public:
   Circle(int r) : Shape(r, r) {}
   void display() {
      cout << "Radius: " << length << endl;
   }
};

int main() {
   Shape s(5, 10);
   Circle c(7);
   s.display();
   c.display();
   return 0;
}

Output:

Length: 5, Width: 10
Radius: 7

Polymorphism and its Roles

Q: What is polymorphism? A: Polymorphism is the ability of an object to take on many forms. In C++, it is achieved through virtual functions.

Q: What are the roles of polymorphism? A: The roles of polymorphism are to allow for code reuse, to simplify program design, and to enable dynamic binding.

Example:

class Shape {
public:
   virtual void draw() {}
};

class Circle : public Shape {
public:
   void draw() {
      cout << "Drawing a Circle" << endl;
   }
};

class Rectangle : public Shape {
public:
   void draw() {
      cout << "Drawing a Rectangle" << endl;
   }
};

void drawShapes(Shape** s, int count) {
   for (int i = 0; i < count; i++) {
      s[i]->draw();
   }
}

int main() {
   Shape* s[2];
   s[0] = new Circle();
   s[1] = new Rectangle();
   drawShapes(s, 2);
   delete s[0];
   delete s[1];
   s[0] = nullptr;
   s[1] = nullptr;
   return 0;
}

Output:

Drawing a Circle
Drawing a Rectangle
